[edit] Requirements (SIA)

1. S/MIME (Secure/Multipurpose Internet Mail Extensions) is a standard for public key encryption and signing of MIME data. Following MS Exchange policies, related to S/MIME, shall be supported:
    • Message signing
    • Message encryption
    • Inclusion of signing algorithms in Encrypted messages
    • Inclusion of encryption algorithms in Encrypted messages
    • Ability to negotiate the encryption algorithm
    • Allowing software certificates for digital signatures
    • Allowing a specific or some specific algorithms for signing
    • Allowing a specific or some specific algorithms for encryption
2. Digitally signed e-mail shall be stored with the digital signature.

3. Users shall have the option to digitally sign all messages, or digitally sign each message manually.

4. Users shall have the capabilities to add, or remove certificates to/from certificate stores used by the e-mail client.

Main Test Scenario:

precondition:

  • all the action should  be done under connection with network.
  • a certificates should be installed in external storage on our device.

1. Use Digital signature

  • user logins email successfully
  • user navigates to settings page
  • user selects Digital signature
  • user back to send an email to check this function valid.

2. Use Encryption

  • user logins email successfully
  • user navigates to email settings page
  • user selects Encryption item
  • user sends an email to check this item valid.
