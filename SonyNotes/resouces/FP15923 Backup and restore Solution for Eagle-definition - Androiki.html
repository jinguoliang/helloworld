[edit] Draft Design Proposal (SIA)

Initial design proposal.

  • For SIA, give enough details to understand system impact and for communication purposes.
    • Make sure you explain how you expect your feature to impact the system. Separate between impact in GITs owned by Android vs. SOMC. Impact should preferably be explained on GIT level and for very large GITs on sub-parts. In case you see options then explain these. Use the <a rel="nofollow" class="external text" href="http://metadoc.sonyericsson.net/login_link.asp?doc=13/0113-LXE108117&rev=latest">Architects guidelines</a> to understand wanted and unwanted impact. Describe if new platform and/or application APIs are to be added. Describe if patches are needed and if possible estimate the size and complexity of those. List if customization will be needed and for what. List contribution proposals. List usage of 3rd party components. Elaborate if this feature will impact our development environment, SACCO, Indus, CS, or PC Software. Elaborate if this feature will increase TTM. Elaborate on the status of Google if they are making improvements in the same area

  • For definition, give more detail to better understand implementation/test effort.

Please make a list of all the gits (and library name in case of large git) that will be modified by this feature, and any external dependencies. For example: platform/packages/Bluetooth
